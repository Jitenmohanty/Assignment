# Section 1 — django-silk query-count evidence

Generated by `python manage.py capture_silk_evidence --orders 200 --items 3`.

The same customer (200 orders, 3 items each) was requested through
both endpoints. Query counts below are the values **django-silk recorded**
(`silk.models.Request.num_sql_queries`) while the requests ran through
`SilkyMiddleware`. Browse them interactively at `/silk/`.

| Endpoint | View | SQL queries |
| --- | --- | --- |
| `GET /api/orders/summary/broken/` | `BrokenOrderSummaryView` | **601** |
| `GET /api/orders/summary/`        | `FixedOrderSummaryView`  | **1** |

**Reduction: 99.8%.**

The broken endpoint's count tracks `1 + 3N` (one base query plus a customer
lookup, a `COUNT(*)`, and an items fetch per order). The fixed endpoint collapses
to a small constant because `select_related("customer")` joins the to-one FK and
`.annotate(Count/Sum)` computes the aggregates in a single `GROUP BY` query.

Re-run this command any time to regenerate the numbers from a clean state.
